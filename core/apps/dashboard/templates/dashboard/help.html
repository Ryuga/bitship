Deployment tutorial

How to deploy your Python bot easily on our platform.




Launch a Python app


Go to the dashboard panel, click launch and select python.

Enter your app name (unique) and select your plan. Once done hit launch. You will be automatically redirected to the app management console




Upload your bot files and recreate the project structure.


Upload your bot files using the upload button, Create folders and recreate your bot project structure exactly as it is on the panel


Don't forget to add the requirement.txt or Pipfile with all the packages listed on your project root directory.


Set app configurations

Press the configure button to set the app configurations, select the python version and main file (file from which code execution starts), Save.



Deploy your app

Now you are all set, click the deploy button to deploy your app and wait for the bot to boot up.



What is requirements.txt?

If you have browsed any python projects on Github or elsewhere, you have probably noticed a file called requirements.txt

This requirements.txt file is used for specifying what python packages are required to run the project you are looking at.
Typically the requirements.txt file is located in the root directory of your project.

If you open a requirements.txt file, you will see something that looks similar to this:

discord.py==1.6.0

aiohttp==3.7.4

async-lru==1.0.2

async-timeout==3.0.1

virtualenv==16.3.0

Your app needs either requirements.txt or Pipfile in the root folder to run.



What is a Pipfile?

Pipfile is the dedicated file used by the Pipenv virtual environment to manage project dependencies.

This file is essential for using Pipenv. When you create a Pipenv environment either for a new or an existing project, the Pipfile is generated automatically.

Your app needs either Pipfile or requirements.txt in the root folder to run.

What is a package.json

All npm packages contain a file, usually in the project root, called package.json - this file holds various metadata relevant to the project.

This file is used to give information to npm that allows it to identify the project as well as handle the project's dependencies.

Your app needs this file in the root folder to run
